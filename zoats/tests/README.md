### `smoke.py` - Fixture Smoke Test

**Purpose:** Validate fixture integrity and smoke-test prerequisites on the `test-fixture` job

**Usage:**
```bash
python3 tests/smoke.py
```

**What it tests:**
- Job contract files exist
- `metadata.json` validates against `schemas/job.schema.json`
- Fixture candidate folders contain raw resumes
- `pipeline_run.json` shape looks valid when present

**Test Job:** `test-fixture`  
**Test Candidates:** `candidate-alpha`, `candidate-beta`

**Exit codes:**
- 0: All tests passed
- 1: One or more tests failed

**Output:**
- Detailed per-candidate validation
- Decision distribution summary
- Pass/fail summary

## Adding New Tests

1. Create test script in `tests/` directory
2. Follow naming convention: `test_<feature>.py`
3. Use logging for output (see smoke.py example)
4. Return proper exit codes (0=pass, 1=fail)
5. Document in this README
